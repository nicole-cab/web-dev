010 Grid

Grid Terminology

  • grid lines: if you have a grid with 4 columns then there is 5 grid column lines, the lines are numbered starting from 1, following the writing-mode and script direction, e.g. if writing is from left to right like english then 1 will be on the left, but it writing is right to left like arabic then 1 will be on the right
  • grid tracks: a track is the space between two grid lines, so a grid row track would be the space between two row lines and a clumn track would be the space between two column tracks (a grid row track is just a row)
  • grid cell: like a cell in a spreadsheet, by default each item is laid out one item into each defined cell
  • grid area: several grid cells together, created by causing an item to span over multiple tracks
  • gaps: space between tracks, you can't place content into a gap
  • grid container: "display: grid"
  • grid item: item which is a direct child of the grid container

intrinsic sizing keywords

  • min-content: track is as small as it can be without overflowing
  • max-content: track becomes as wide enough for all of the content to display in one long unbroken string - can cause overflow as string will not wrap
  • fit-content(): acts like a max-content at first, however once the track reaches the size you pass into the function, the content starts to wrap

- auto sized tracks are stretched to fit the container by default

the fr unit

- you can use fr to space out items using fractions

the minmax() function

- minmax(min_size, max_size) is used to set the min and max size for a track

- fr units can be used inside the minmax() function

- to force a track to take an equal share of the space in the grid container minus gaps use minmax, use minmax(0, 1fr), this makes the minimum size of the track 0 and not the min-content size

Auto-placement

- auto-placement: one element/item per cell

placing items in columns

- the default behaviour of grid layout is to place the items along the rows, you can instead cause the items to place into columns using "grid-auto-flow: column", you need to define row tracks or else the items will create column tracks and layout out all in one long row

- grid-auto-flow: row; runs in the direction of the writing mode e.g. in english we write from left to right

spanning tracks

- you can cause some or all of the items in an auto-placed layout to span more than one track:

  • "grid-column-start: number | span n | auto" to define what column the item starts
    number = column it starts
    span n = number of columns the item should span/take up
    auto = follows normal flow which is one item per cell
  • "grid-column-end: number | span a | auto" to define what column the item ends
    number = column it ends
    span n = number of columns the item should span/take up
    auto = 1 column per item
  • "grid-column: grid-column-start_value / grid-column-end_value" is the shorthand version

filling gaps

- the default behaviour of grid is to fill up a track as much as possible before moving on to the next track, so if you span items across many columns this may cause gaps to be created, one way to make good use of space is by using "grid-auto-flow: dense" which will try to put smaller items into the gaps to fill it in, this means that the order of the items may not follow the correct order

stacking items

- you can make items overlap each other in grids when spanning them across columns, you can use the z-index property to place an item on top of the other, the higher number the more items it will cover

Named grid lines

- instead of refering to track lines using numbers, you can name them by declaring the name of the line inside square brackets before defining the size of the column: "grid-template-columns: [line-one] 1fr [line-two] 3fr [line-three];" this would make a layout with 2 columns and 3 track lines

- you can also use multiple names for a track line: [name-one name-two]

Grid Template Areas

- you can name grid areas by naming the direct children/items of the grid container using "grid-area: name-goes-here;" then using "grid-template-areas" in the grid container to define which grid cells each item will span e.g. the children are named, header, sidebar, footer and the areas are the defined:

  • grid-template-areas:
    "header header header header"
    "sidebar content content content"
    "sidebar footer footer footer"
  • this would create a layout with 4 columns and 3 rows with 4 areas spanning across the corresponding cells
  • area names can be anything you like other than the keywords "auto" and "span"
  • grid must have no empty cells
  • the areas have to be rectangular and cannot be disconnected
  • to leave white space in the grid use a "." or multiples with no white space between them e.g.
    • grid-template-areas:
      "....... header header header"
      "sidebar sidebar content content"
      "sidebar footer footer footer"
    • remember that grid-template-areas relates to writing-mode and direction

Shorthand Properties

grid-template

- grid-template is shorthand for grid-template-rows, grid-template-columns and grid-template-areas

- rows are defined first along with the value of grid-template-areas, column sizing is added after a /

  • grid-template:
    "head head head" minmax(150px, auto)
    "sidebar content content" auto
    "sidebar footer footer" auto / 1fr 1fr 1fr;

grid property

- the grid property is shorthand for:

  • grid-template-rows
  • grid-template-columns
  • grid-template-areas
  • grid-auto-rows
  • grid-auto-columns
  • grid-auto-flow

Alignment

- justify-content and align-content distribute additional space in the grid container around or between tracks

- justify-self and align-self are applied to a grid item to move it around inside the grid area it is placed in

- justify-tems and align-items are applied to the grid container to set all of the justify-self properties ont he items
