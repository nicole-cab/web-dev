wrapping flex items

- the initial value of flex-wrap: nowrap, meaning if there isn't enough space in the container the items will overflow, items will shrink as small as they can, down to the min-content size before overflow happens

- to wrap use flex-wrap: wrap, when a flex container wraps it creates multiple flex lines, each flex line acts as its own flex container therefore you can't line up something in row 2 with something in row1

- you can set both flex-direction and flex-wrap in shorthand using flex-flow, e.g. "flex-flow: column wrap;"

controlling space inside flex items

- given that there has more spacen in the flex container after the items are displayed, the items will not grow to fill this space, they stop growing at their max-content size, this is because the initial values of the flex- properties is:

  • flex grow: 0; items don't grow
  • flex-shrink: 1; items can shrink smaller than their flex-basis
  • flex-basis: auto; items have a base size of auto

- the above list can be represented using flex: initial

- flex: auto; makes the items grow (up to their max-content size), allowing large items to have more space than small ones, flex: auto has the following properties

  • flex-grow: 1; items can grow larger than their flex-basis
  • flex-shrink: 1; items can shrink smaller than their flex-basis
  • flex-basis: auto; items have a base size of auto

- width: max-content | min-content | fit-content

  • max-content: ideal size if there was unlimited space
  • min-content: min size the box could take without overflowing
  • fit-content: mix of max and min-content, if there is enough space use max-content, if not use min-content

- to make all the items have consistent size use "flex: 1" which is equivalent to:

  • flex-grow: 1; all items grow equally
  • flex-shrink: 1
  • flex-basis: 0; all items have 0 size therefore all of the space in the flex container is available to be distributed

- flex: none; items don't shrink or grow, this is purely used to access the alignment properties of flexbox without the flexible behaviour

reordering flex items

- items can be ordered using the order property, for e.g. if you want a specific item to be placed 3rd in a row you can use order: 3, if multiple items have the same order then they will be displayed together the first element defined going first

- it is not recommended to the order property as it can cause some problems, change your logical order in html instead!

flexbox alignment overview

properties for space distribution

  • justify-content: space distribution on the main axis
  • align-content: space distribution on the cross axis
  • place-content: shorthand for setting both

properties for alignment

  • align-self: aligns a single item on the cross axis
  • align-items: aligns all the items as a group on the cross axis

distributing space on the main axis

- you can use the following properties

  • - for the main axis use "justify-content: flex-start | flex-end | space-between | center | space-around | space-evenly | initial | inherit"
  • - for the cross axis use the same property but you need to specify the height of the container to have space, you don't need to do this horizontally since divs span across the entire line

distributing space between flex lines

- you can use align-content with the same values as justify-content to distribute the flex lines on the cross axis, the initial value of align-content is stretch, (align-content is like justify-content but for flex lines)

- place-content is shorthand for justify-content and align-content, if you only specify one value then it applies it to both, if two values, then the first value is align-content, second is justify-content

aligning items on the cross-axis

- in the cross axis you can align items within the flex line using align-items and align-self

- the initial value of align-self is stretch which makes the all the items stretch to the height of the tallest item by default

- align-item has the following values:

  • flex-start
  • flex-end
  • center
  • stretch
  • baseline
